# Contributing

Thanks for your interest in `renovate-mcp`. Before opening a non-trivial PR, please skim [`CLAUDE.md`](./CLAUDE.md) — it documents the architectural invariants this repo cares about (when to shell out to the Renovate CLI vs. stay in-process, the `write_config` validate-then-rename flow, how the preset catalogue is generated, what `preview_custom_manager` deliberately does *not* implement, and so on). Most review feedback lands on those invariants, so reading them first will save a round trip. The roadmap for what's intentionally left out of v1 lives in the [open GitHub issues](https://github.com/tibuntu/renovate-mcp/issues) — pick one of those if you're looking for something to work on, and read its scope/acceptance criteria before starting.

A few conventions worth knowing up front: commits use [Conventional Commits](https://www.conventionalcommits.org/) (release-please drives versioning and `CHANGELOG.md` from the prefixes — see the [Release flow](./README.md#release-flow) section in the README). Run `npm run typecheck` and `npm test` before pushing; CI runs both on Node 24 and will reject anything that doesn't pass. For security issues, please follow [`SECURITY.md`](./SECURITY.md) rather than opening a public issue.
